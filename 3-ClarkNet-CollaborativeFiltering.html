Generate integer IDs for hosts and requests¶

Since pyspark methods work with integer IDs for "host" ("user") and "request" ("item"), I generate those

Technical challenges and details to overcome those¶

The problem is to create an integer id for each host (and then each request). I need integer because this is the only type that ALS algorithm accepts.

This spark DataFrame unique id can be done created with pyspark.sql.functionsmonotonicallyincreasing_id(). It generates, however, the 64-bit _long type in the whole range of possible values, so that there is no way to conver these to 32-bit integer with garantie of keeping the unique property. This is rather inconsistency inside spark which hopefully will be fixed sometime.

The way around that I use here is through RDD using zipWithIndex. Another way through pandas data frame (if hosts and their ids fit in memory) is also possible

Train / validation / test split¶

I use date-time column to split dataset into train and test. The last day (Sep 10) will go to test, last but one (Sep 9) to validation and the previous 5 days will be used for training. After splitting date_time column is not needed anymore, since the recommendation system discussed here do not use time development.

Prepare DataFrame for training with CF model. Rename columns as "user" and "item" and add a "rating" column filled with "1"s (indicating the user issued item) as implicit feedback.

In [8]:
def prepare_DF_for_CFmodel(dataFrame, train_items_DF=train_items_DF, train_users_DF=train_users_DF):
    """
    Prepares data frame for using with collaborative filtering model
    --- keeps only users from train_users_DF (hosts with more than 10 unique requests)
    --- keeps only items from train_items_DF (requests issued by more than 10 unique hosts)
    --- calculates "rating" as log(1+count)

    Input: 
        * dataFrame - spark DataFrame of (user, item, count)
        * train_items_DF - requests issued by more than 10 unique hosts  
        * train_users_DF - hosts with more than 10 unique requests
        
    Output: 
        * spark DataFrame of (user, item, rating)
    """

    forCF_DF = (dataFrame.join(func.broadcast(train_items_DF), "item")
                       .join(func.broadcast(train_users_DF), "user")
                       #.dropDuplicates().withColumn("rating", func.lit(float(1))) 
                       #.groupBy("user", "item").count().withColumnRenamed("count", "rating")
                       .groupBy("user", "item").count().withColumn("rating", func.log1p("count")).drop("count"))
                       
    return forCF_DF

Evaluation with Ranking Metrics¶

By the construction of the algorithm, ALS internally minimizes MSE. This is, however, not appropriate metric for collaborative filtering. The recommendation tast by its nature is a ranking task. Therefore, I use ranking metrics.

First I convert Data Frame with predictions to RDD suitable for usage with RankingMetrics implemented in pyspark.

In [14]:
def prepare_predictionDF_forRankingMetrics(predictionDF):
    """
    Converts prediction DataFrame (obtained from als_model.transform(dataset))
    to RDD suitable for calculating Ranking Metrics
    """
    
    #TODO: intelligent threshold
    
    from operator import itemgetter
    
    foo = (predictionDF.filter("prediction > 0.4").rdd  
                 .map(lambda rec: (rec.user, [(rec.item, rec.rating, rec.prediction)]))
                 .reduceByKey(lambda p,q: p+q)
                 .mapValues(lambda v: ( [g[0] for g in sorted(v, key=itemgetter(2), reverse=True)],   
                                        [g[0] for g in sorted(v, key=itemgetter(1), reverse=True)]  ) )
                 .values()
          )
    return foo
